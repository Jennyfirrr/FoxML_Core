### Color Scheme (Terminal-Friendly)
- **Green**: Success, active, healthy status
- **Yellow**: Warning, in-progress, caution
- **Red**: Error, critical, stopped
- **Blue**: Information, neutral status
- **Gray**: Disabled, queued, pending

### Interaction
- **Keyboard-first**: All actions via keyboard shortcuts
- **No mouse required**: Works in any terminal
- **Fast updates**: Smooth 60fps refresh for real-time data
- **Auto-scroll**: Event logs auto-scroll with option to pause
- **Filtering**: Filter events by type, severity

### Layout Philosophy
- **Information density**: Show maximum useful info without clutter
- **Progressive disclosure**: Details available on demand
- **Contextual**: Show what's relevant to current view
- **Responsive**: Adapts to terminal size (minimum 80x24)
